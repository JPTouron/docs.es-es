### <a name="exception-message-has-changed-for-failed-datacontract-serialization-in-case-of-an-unknown-type"></a>Ha cambiado el mensaje de excepción para el error de serialización de DataContract en el caso de un tipo desconocido

#### <a name="details"></a>Detalles

A partir de .NET Framework 4.6, se ha clarificado el mensaje de excepción dado si no se puede serializar o deserializar <xref:System.Runtime.Serialization.DataContractSerializer?displayProperty=fullName> o <xref:System.Runtime.Serialization.Json.DataContractJsonSerializer?displayProperty=fullName> debido a que faltan "tipos conocidos".

#### <a name="suggestion"></a>Sugerencia

Las aplicaciones no deben depender de mensajes de excepción específicos. Si una aplicación depende de este mensaje, se puede actualizar para que espere el mensaje nuevo o, preferiblemente, cambiarla para que solo dependa del tipo de excepción.

|:--------|:------------|
| Ámbito   |Borde|
|Versión|4.6|
|Tipo|Tiempo de ejecución
