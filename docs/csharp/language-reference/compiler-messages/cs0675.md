# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="bc18a-103">Advertencia del compilador (nivel 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="bc18a-103">Compiler Warning (level 3) CS0675</span></span>

<span data-ttu-id="bc18a-104">Operador OR bit a bit usado en un operando con extensión de signo; considere la posibilidad de convertir primero a un tipo sin signo más pequeño</span><span class="sxs-lookup"><span data-stu-id="bc18a-104">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="bc18a-105">El compilador ha ampliado y extendido el signo de una variable implícitamente, y después ha usado el valor resultante en una operación OR bit a bit.</span><span class="sxs-lookup"><span data-stu-id="bc18a-105">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="bc18a-106">Esto puede dar como resultado un comportamiento inesperado.</span><span class="sxs-lookup"><span data-stu-id="bc18a-106">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="bc18a-107">En el ejemplo siguiente se genera la advertencia CS0675:</span><span class="sxs-lookup"><span data-stu-id="bc18a-107">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
