# <a name="compiler-error-cs0311"></a><span data-ttu-id="135a2-103">Error del compilador CS0311</span><span class="sxs-lookup"><span data-stu-id="135a2-103">Compiler Error CS0311</span></span>

<span data-ttu-id="135a2-104">El tipo "type1" no se puede usar como parámetro de tipo "T" en el tipo o método genérico "\<name>".</span><span class="sxs-lookup"><span data-stu-id="135a2-104">The type 'type1' cannot be used as type parameter 'T' in the generic type or method '\<name>'.</span></span> <span data-ttu-id="135a2-105">No hay ninguna conversión de referencia implícita de "type1" a "type2".</span><span class="sxs-lookup"><span data-stu-id="135a2-105">There is no implicit reference conversion from 'type1' to 'type2'.</span></span>  
  
 <span data-ttu-id="135a2-106">Cuando se aplica una restricción a un parámetro de tipo genérico, debe existir una conversión de referencia o identidad implícita del argumento concreto al tipo de la restricción.</span><span class="sxs-lookup"><span data-stu-id="135a2-106">When a constraint is applied to a generic type parameter, an implicit identity or reference conversion must exist from the concrete argument to the type of the constraint.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="135a2-107">Para corregir este error</span><span class="sxs-lookup"><span data-stu-id="135a2-107">To correct this error</span></span>  
  
1. <span data-ttu-id="135a2-108">Cambie el argumento utilizado para crear la clase.</span><span class="sxs-lookup"><span data-stu-id="135a2-108">Change the argument you are using to create the class.</span></span>  
  
2. <span data-ttu-id="135a2-109">Si es propietario de la clase, puede quitar la restricción o bien hacer algo para habilitar una conversión de identidad o de referencia implícita.</span><span class="sxs-lookup"><span data-stu-id="135a2-109">If you own the class, you can remove the constraint or else do something to enable an implicit reference or identity conversion.</span></span> <span data-ttu-id="135a2-110">Por ejemplo, puede hacer que el segundo tipo herede del primero.</span><span class="sxs-lookup"><span data-stu-id="135a2-110">For example, you can make the second type inherit from the first.</span></span>  
  
## <a name="example"></a><span data-ttu-id="135a2-111">Ejemplo</span><span class="sxs-lookup"><span data-stu-id="135a2-111">Example</span></span>  
  
```csharp  
// cs0311.cs  
class B {}  
class C {}  
class Test<T> where T : C  
{ }  
  
class Program  
{  
    static void Main()  
    {  
        Test<B> test = new Test<B>(); //CS0311  
    }  
}  
```  
  
 <span data-ttu-id="135a2-112">Si este error se produce al intentar utilizar un argumento de tipo de valor, tenga en cuenta que una conversión numérica implícita, por ejemplo, de `short` a `int`, no satisface un parámetro de tipo genérico.</span><span class="sxs-lookup"><span data-stu-id="135a2-112">If this error occurs when trying to use a value-type argument, notice that an implicit numeric conversion, for example from `short` to `int`, does not satisfy a generic type parameter.</span></span>
