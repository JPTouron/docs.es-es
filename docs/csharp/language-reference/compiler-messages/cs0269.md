# <a name="compiler-error-cs0269"></a><span data-ttu-id="2844a-103">Error del compilador CS0269</span><span class="sxs-lookup"><span data-stu-id="2844a-103">Compiler Error CS0269</span></span>
<span data-ttu-id="2844a-104">Uso del parámetro out sin asignar "parameter"</span><span class="sxs-lookup"><span data-stu-id="2844a-104">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="2844a-105">El compilador no pudo comprobar que el parámetro out se hubiera asignado a un valor antes de usarlo; es posible que su valor no estuviera definido al asignarlo.</span><span class="sxs-lookup"><span data-stu-id="2844a-105">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="2844a-106">Asegúrese de asignar un valor a los parámetros `out` en el método al que se ha llamado antes de tener acceso al valor.</span><span class="sxs-lookup"><span data-stu-id="2844a-106">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="2844a-107">Si necesita usar el valor de la variable que se ha pasado, use un parámetro `ref` en su lugar.</span><span class="sxs-lookup"><span data-stu-id="2844a-107">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="2844a-108">Para obtener más información, vea [Pasar parámetros (Guía de programación de C#)](../../programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="2844a-108">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="2844a-109">Ejemplo</span><span class="sxs-lookup"><span data-stu-id="2844a-109">Example</span></span>  
 <span data-ttu-id="2844a-110">En el ejemplo siguiente se genera el error CS0269:</span><span class="sxs-lookup"><span data-stu-id="2844a-110">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="2844a-111">Ejemplo</span><span class="sxs-lookup"><span data-stu-id="2844a-111">Example</span></span>  
 <span data-ttu-id="2844a-112">También se puede producir si la inicialización de una variable tiene lugar en un bloque try y el compilador no puede comprobar si este se ejecutará de manera correcta:</span><span class="sxs-lookup"><span data-stu-id="2844a-112">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
