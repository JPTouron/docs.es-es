# <a name="compiler-error-cs0571"></a><span data-ttu-id="98361-103">Error del compilador CS0571</span><span class="sxs-lookup"><span data-stu-id="98361-103">Compiler Error CS0571</span></span>

<span data-ttu-id="98361-104">'function': no se puede llamar explícitamente al operador o al descriptor de acceso</span><span class="sxs-lookup"><span data-stu-id="98361-104">'function' : cannot explicitly call operator or accessor</span></span>  
  
 <span data-ttu-id="98361-105">Ciertos operadores tienen nombres internos.</span><span class="sxs-lookup"><span data-stu-id="98361-105">Certain operators have internal names.</span></span> <span data-ttu-id="98361-106">Por ejemplo, **op_Increment** es el nombre interno del operador ++.</span><span class="sxs-lookup"><span data-stu-id="98361-106">For example, **op_Increment** is the internal name of the ++ operator.</span></span> <span data-ttu-id="98361-107">No se debe usar o llamar explícitamente a dichos nombres de método.</span><span class="sxs-lookup"><span data-stu-id="98361-107">You should not use or explicitly call such method names.</span></span>  
  
 <span data-ttu-id="98361-108">En el ejemplo siguiente se genera la advertencia CS0571:</span><span class="sxs-lookup"><span data-stu-id="98361-108">The following sample generates CS0571:</span></span>  
  
```csharp  
// CS0571.cs  
public class MyClass  
{  
   public static MyClass operator ++ (MyClass c)  
   {  
      return null;  
   }  
  
   public static int prop  
   {  
      get  
      {  
         return 1;  
      }  
      set  
      {  
      }  
   }  
  
   public static void Main()  
   {  
      op_Increment(null);   // CS0571  
      // use the increment operator as follows  
      // MyClass x = new MyClass();  
      // x++;  
  
      set_prop(1);      // CS0571  
      // try the following line instead  
      // prop = 1;  
   }  
}  
```
