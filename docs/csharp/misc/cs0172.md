# <a name="compiler-error-cs0172"></a><span data-ttu-id="d60ee-103">Error del compilador CS0172</span><span class="sxs-lookup"><span data-stu-id="d60ee-103">Compiler Error CS0172</span></span>

<span data-ttu-id="d60ee-104">No se puede determinar el tipo de expresión condicional porque 'type1' y 'type2' se convierten implícitamente uno en el otro</span><span class="sxs-lookup"><span data-stu-id="d60ee-104">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="d60ee-105">En una instrucción condicional, debe poder convertir implícitamente los tipos en cualquier lado del `:` token.</span><span class="sxs-lookup"><span data-stu-id="d60ee-105">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="d60ee-106">Además, no puede haber conversiones explícitas mutuas; solo se necesita una conversión.</span><span class="sxs-lookup"><span data-stu-id="d60ee-106">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="d60ee-107">El ejemplo siguiente genera la advertencia CS0172:</span><span class="sxs-lookup"><span data-stu-id="d60ee-107">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
