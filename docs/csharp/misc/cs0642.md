<span data-ttu-id="2847d-104">Posible instrucción vacía errónea</span><span class="sxs-lookup"><span data-stu-id="2847d-104">Possible mistaken empty statement</span></span>  
  
 <span data-ttu-id="2847d-105">Un punto y coma después de una instrucción condicional puede hacer que su código se ejecute de forma diferente a lo esperado.</span><span class="sxs-lookup"><span data-stu-id="2847d-105">A semicolon after a conditional statement may cause your code to execute differently than intended.</span></span>  
  
 <span data-ttu-id="2847d-106">Puede usar la opción del compilador **/nowarn** o `#pragmas warning` para deshabilitar esta advertencia; vea [-nowarn (opciones del compilador de C#)](../language-reference/compiler-options/nowarn-compiler-option.md) o [#pragma ADVERTENCIA](../language-reference/preprocessor-directives/preprocessor-pragma-warning.md) para obtener más información.</span><span class="sxs-lookup"><span data-stu-id="2847d-106">You can use **/nowarn** compiler option or `#pragmas warning` to disable this warning; see [-nowarn (C# Compiler Options)](../language-reference/compiler-options/nowarn-compiler-option.md) or [#pragma warning](../language-reference/preprocessor-directives/preprocessor-pragma-warning.md) for more information.</span></span>  
  
 <span data-ttu-id="2847d-107">El ejemplo siguiente genera la advertencia CS0642:</span><span class="sxs-lookup"><span data-stu-id="2847d-107">The following sample generates CS0642:</span></span>  
  
```csharp  
// CS0642.cs  
// compile with: /W:3  
class MyClass  
{  
   public static void Main()  
   {  
      int i;  
  
      for (i = 0; i < 10; i += 1);   // CS0642 semicolon intentional?  
      {  
         System.Console.WriteLine (i);  
      }  
   }  
}  
```
