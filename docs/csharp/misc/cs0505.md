<span data-ttu-id="6300d-104">'miembro1': no se puede reemplazar porque 'miembro2' no es una función.</span><span class="sxs-lookup"><span data-stu-id="6300d-104">'member1': cannot override because 'member2' is not a function</span></span>  
  
 <span data-ttu-id="6300d-105">Una declaración de clase intentó reemplazar un elemento que no es un método en una clase base.</span><span class="sxs-lookup"><span data-stu-id="6300d-105">A class declaration attempted to override a non-method in a base class.</span></span> <span data-ttu-id="6300d-106">Los reemplazos deben coincidir en el tipo de miembro.</span><span class="sxs-lookup"><span data-stu-id="6300d-106">Overrides must match the member type.</span></span> <span data-ttu-id="6300d-107">Si quiere un método con el mismo nombre que un método en una clase base, use [new](../language-reference/keywords/new-modifier.md) (y no [override](../language-reference/keywords/override.md)) en la declaración del método en la clase base.</span><span class="sxs-lookup"><span data-stu-id="6300d-107">If a method with the same name as a method in a base class is desired, use [new](../language-reference/keywords/new-modifier.md) (and not [override](../language-reference/keywords/override.md)) on the method declaration in the base class.</span></span>  
  
 <span data-ttu-id="6300d-108">El ejemplo siguiente genera la advertencia CS0505:</span><span class="sxs-lookup"><span data-stu-id="6300d-108">The following sample generates CS0505:</span></span>  
  
```csharp  
// CS0505.cs  
// compile with: /target:library  
public class clx  
{  
   public int i;  
}  
  
public class cly : clx  
{  
   public override int i() { return 0; }   // CS0505  
}  
```
