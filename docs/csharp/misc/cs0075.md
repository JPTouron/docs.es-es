# <a name="compiler-error-cs0075"></a><span data-ttu-id="b481a-103">Error del compilador CS0075</span><span class="sxs-lookup"><span data-stu-id="b481a-103">Compiler Error CS0075</span></span>

<span data-ttu-id="b481a-104">Para convertir un valor negativo, el valor debe ir entre paréntesis</span><span class="sxs-lookup"><span data-stu-id="b481a-104">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="b481a-105">Si realiza la conversión mediante una palabra clave que identifica un tipo predefinido, no son necesarios los paréntesis.</span><span class="sxs-lookup"><span data-stu-id="b481a-105">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="b481a-106">De lo contrario, debe colocar los paréntesis porque (x) – y no se considerará una expresión de conversión.</span><span class="sxs-lookup"><span data-stu-id="b481a-106">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="b481a-107">De la especificación C#, sección 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="b481a-107">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="b481a-108">*En la regla de desambiguación se sigue que, si x e y son identificadores, (x) y, (x) (y) y (x) (-y) son expresiones de conversión, pero (x)-y no, aunque x identifique un tipo. Sin embargo, si x es una palabra clave que identifica un tipo predefinido (como int), las cuatro formas son expresiones de conversión (porque tal palabra clave podría no ser una expresión por sí misma).*</span><span class="sxs-lookup"><span data-stu-id="b481a-108">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="b481a-109">El código siguiente genera el error CS0075:</span><span class="sxs-lookup"><span data-stu-id="b481a-109">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
