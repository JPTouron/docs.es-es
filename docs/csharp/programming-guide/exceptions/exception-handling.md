## <a name="finally-blocks"></a><span data-ttu-id="1399c-126">Bloques Finally</span><span class="sxs-lookup"><span data-stu-id="1399c-126">Finally Blocks</span></span>  

 <span data-ttu-id="1399c-127">Los bloques `finally` permiten limpiar las acciones que se realizan en un bloque `try`.</span><span class="sxs-lookup"><span data-stu-id="1399c-127">A `finally` block enables you to clean up actions that are performed in a `try` block.</span></span> <span data-ttu-id="1399c-128">Si está presente, el bloque `finally` se ejecuta en último lugar, después del bloque `try` y de cualquier bloque `catch` coincidente.</span><span class="sxs-lookup"><span data-stu-id="1399c-128">If present, the `finally` block executes last, after the `try` block and any matched `catch` block.</span></span> <span data-ttu-id="1399c-129">Los bloques `finally` siempre se ejecutan, independientemente de si se produce una excepción o si se encuentra un bloque `catch` que coincida con el tipo de excepción.</span><span class="sxs-lookup"><span data-stu-id="1399c-129">A `finally` block always runs, regardless of whether an exception is thrown or a `catch` block matching the exception type is found.</span></span>  
  
 <span data-ttu-id="1399c-130">Los bloques `finally` pueden usarse para liberar recursos como secuencias de archivo, conexiones de base de datos y controladores de gráficos sin necesidad de esperar a que el recolector de elementos no utilizados en tiempo de ejecución finalice los objetos.</span><span class="sxs-lookup"><span data-stu-id="1399c-130">The `finally` block can be used to release resources such as file streams, database connections, and graphics handles without waiting for the garbage collector in the runtime to finalize the objects.</span></span> <span data-ttu-id="1399c-131">Consulte [using (Instrucción)](../../language-reference/keywords/using-statement.md) para obtener más información.</span><span class="sxs-lookup"><span data-stu-id="1399c-131">See [using Statement](../../language-reference/keywords/using-statement.md) for more information.</span></span>
