 <span data-ttu-id="72bc7-123">Este esquema de nomenclatura facilita a los desarrolladores de bibliotecas la tarea de extender .NET Framework para crear grupos jerárquicos de tipos y asignarles nombre de forma coherente e informativa.</span><span class="sxs-lookup"><span data-stu-id="72bc7-123">This naming scheme makes it easy for library developers extending the .NET Framework to create hierarchical groups of types and name them in a consistent, informative manner.</span></span> <span data-ttu-id="72bc7-124">También permite identificar de forma inequívoca los tipos mediante su nombre completo (es decir, por su espacio de nombres y nombre de tipo), lo que evita que se produzcan conflictos entre los nombres de tipo.</span><span class="sxs-lookup"><span data-stu-id="72bc7-124">It also allows types to be unambiguously identified by their full name (that is, by their namespace and type name), which prevents type name collisions.</span></span> <span data-ttu-id="72bc7-125">Se espera que los programadores de bibliotecas usen la siguiente convención cuando creen nombres para sus propios espacios de nombres:</span><span class="sxs-lookup"><span data-stu-id="72bc7-125">Library developers are expected to use the following convention when creating names for their namespaces:</span></span>
