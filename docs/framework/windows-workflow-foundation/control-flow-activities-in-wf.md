<span data-ttu-id="e11ba-104">[!INCLUDE[netfx_current_long](../../../includes/netfx-current-long-md.md)] proporciona varias actividades para controlar el flujo de ejecución dentro de un flujo de trabajo.</span><span class="sxs-lookup"><span data-stu-id="e11ba-104">The [!INCLUDE[netfx_current_long](../../../includes/netfx-current-long-md.md)] provides several activities for controlling flow of execution within a workflow.</span></span> <span data-ttu-id="e11ba-105">Algunas de estas actividades (como `Switch` y `If` ) implementan estructuras de control de flujo similares a las de los entornos de programación como Visual C#, mientras que otras (como `Pick` ) modelan nuevas estructuras de programación.</span><span class="sxs-lookup"><span data-stu-id="e11ba-105">Some of these activities (such as `Switch` and `If`) implement flow control structures similar to those in programming environments such as Visual C#, while others (such as `Pick`) model new programming structures.</span></span>  
  
 <span data-ttu-id="e11ba-106">Tenga en cuenta que mientras las actividades `Parallel` y `ParallelForEach` programan varias actividades secundarias para su ejecución simultánea, solo se usa un child activity of these activities executes sequentially and successive activities do not execute until previous activities either complete or go idle.</span></span> <span data-ttu-id="e11ba-108">Como data-stu-id="e11ba-108">As a result, these activities are most useful for applications in which several potentially blocking activities must execute in an interleaved fashion.</span></span> <span received or its asynchronous work to be completed.</span></span>
