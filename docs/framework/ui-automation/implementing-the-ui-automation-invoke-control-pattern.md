## <a name="implementation-guidelines-and-conventions"></a><span data-ttu-id="6ec12-112">Directrices y convenciones de implementación</span><span class="sxs-lookup"><span data-stu-id="6ec12-112">Implementation Guidelines and Conventions</span></span>

<span data-ttu-id="6ec12-113">Al implementar el patrón de control Invoke, tenga en cuenta las siguientes directrices y convenciones:</span><span class="sxs-lookup"><span data-stu-id="6ec12-113">When implementing the Invoke control pattern, note the following guidelines and conventions:</span></span>

- <span data-ttu-id="6ec12-114">Los controles implementan <xref:System.Windows.Automation.Provider.IInvokeProvider> si el mismo comportamiento no se expone a través de otro proveedor de patrón de control.</span><span class="sxs-lookup"><span data-stu-id="6ec12-114">Controls implement <xref:System.Windows.Automation.Provider.IInvokeProvider> if the same behavior is not exposed through another control pattern provider.</span></span> <span data-ttu-id="6ec12-115">Por ejemplo, si el método <xref:System.Windows.Automation.InvokePattern.Invoke%2A> de un control realiza la misma acción que los métodos <xref:System.Windows.Automation.ExpandCollapsePattern.Expand%2A> o <xref:System.Windows.Automation.ExpandCollapsePattern.Collapse%2A> , el control no debe implementar <xref:System.Windows.Automation.Provider.IInvokeProvider>.</span><span class="sxs-lookup"><span data-stu-id="6ec12-115">For example, if the <xref:System.Windows.Automation.InvokePattern.Invoke%2A> method on a control performs the same action as the <xref:System.Windows.Automation.ExpandCollapsePattern.Expand%2A> or <xref:System.Windows.Automation.ExpandCollapsePattern.Collapse%2A> method, the control should not implement <xref:System.Windows.Automation.Provider.IInvokeProvider>.</span></span>

- <span data-ttu-id="6ec12-116">Generalmente, la invocación de un control se realiza con un clic, un doble clic, presionando la tecla ENTRAR, usando un método abreviado de teclado predefinido o alguna combinación alternativa de pulsaciones de teclas.</span><span class="sxs-lookup"><span data-stu-id="6ec12-116">Invoking a control is generally performed by clicking or double-clicking or pressing ENTER, a predefined keyboard shortcut, or some alternate combination of keystrokes.</span></span>

- <span data-ttu-id="6ec12-117">Se genera<xref:System.Windows.Automation.InvokePatternIdentifiers.InvokedEvent> en un control que se ha activado (como respuesta a un control que lleva a cabo su acción asociada).</span><span class="sxs-lookup"><span data-stu-id="6ec12-117"><xref:System.Windows.Automation.InvokePatternIdentifiers.InvokedEvent> is raised on a control that has been activated (as a response to a control carrying out its associated action).</span></span> <span data-ttu-id="6ec12-118">Si es posible, se debe generar el evento después de que el control haya completado la acción y haya hecho la devolución sin bloquearse.</span><span class="sxs-lookup"><span data-stu-id="6ec12-118">If possible, the event should be raised after the control has completed the action and returned without blocking.</span></span> <span data-ttu-id="6ec12-119">El evento Invoked debe generarse antes de atender la solicitud Invoke en los escenarios siguientes:</span><span class="sxs-lookup"><span data-stu-id="6ec12-119">The Invoked event should be raised before servicing the Invoke request in the following scenarios:</span></span>

  - <span data-ttu-id="6ec12-120">No es posible ni práctico esperar hasta que se complete la acción.</span><span class="sxs-lookup"><span data-stu-id="6ec12-120">It is not possible or practical to wait until the action is complete.</span></span>

  - <span data-ttu-id="6ec12-121">La acción requiere la interacción del usuario.</span><span class="sxs-lookup"><span data-stu-id="6ec12-121">The action requires user interaction.</span></span>

  - <span data-ttu-id="6ec12-122">La acción requiere mucho tiempo y provocará que el cliente que llama se bloquee durante un tiempo considerable.</span><span class="sxs-lookup"><span data-stu-id="6ec12-122">The action is time-consuming and will cause the calling client to block for a significant amount of time.</span></span>

- <span data-ttu-id="6ec12-123">Si la invocación del control tiene efectos secundarios significativos, esos efectos secundarios se deben exponer a través de la propiedad <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> .</span><span class="sxs-lookup"><span data-stu-id="6ec12-123">If invoking the control has significant side-effects, those side-effects should be exposed through the <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> property.</span></span> <span data-ttu-id="6ec12-124">Por ejemplo, aunque <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> no se asocie a la selección, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> puede provocar que se seleccione otro control.</span><span class="sxs-lookup"><span data-stu-id="6ec12-124">For example, even though <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> is not associated with selection, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> may cause another control to become selected.</span></span>

- <span data-ttu-id="6ec12-125">Normalmente, los efectos de mantener el puntero (o pasar el mouse por encima) no constituyen un evento Invoked.</span><span class="sxs-lookup"><span data-stu-id="6ec12-125">Hover (or mouse-over) effects generally do not constitute an Invoked event.</span></span> <span data-ttu-id="6ec12-126">Sin embargo, los controles que realizan una acción (en lugar de producir un efecto visual) según el estado del efecto de mantener el puntero deben admitir el patrón de control <xref:System.Windows.Automation.InvokePattern> .</span><span class="sxs-lookup"><span data-stu-id="6ec12-126">However, controls that perform an action (as opposed to cause a visual effect) based on the hover state should support the <xref:System.Windows.Automation.InvokePattern> control pattern.</span></span>

> [!NOTE]
> <span data-ttu-id="6ec12-127">Esta implementación se considera un problema de accesibilidad si el control solo se puede invocar como resultado de un efecto secundario relacionado con el mouse.</span><span class="sxs-lookup"><span data-stu-id="6ec12-127">This implementation is considered an accessibility issue if the control can be invoked only as a result of a mouse-related side effect.</span></span>

- <span data-ttu-id="6ec12-128">La invocación de un control es diferente de la selección de un elemento.</span><span class="sxs-lookup"><span data-stu-id="6ec12-128">Invoking a control is different from selecting an item.</span></span> <span data-ttu-id="6ec12-129">No obstante, dependiendo del control, su invocación puede provocar que el elemento se seleccione como un efecto secundario.</span><span class="sxs-lookup"><span data-stu-id="6ec12-129">However, depending on the control, invoking it may cause the item to become selected as a side-effect.</span></span> <span data-ttu-id="6ec12-130">Por ejemplo, la invocación de un elemento de lista de documentos de Microsoft Word en la carpeta Mis documentos selecciona el elemento y abre el documento.</span><span class="sxs-lookup"><span data-stu-id="6ec12-130">For example, invoking a Microsoft Word document list item in the My Documents folder both selects the item and opens the document.</span></span>

- <span data-ttu-id="6ec12-131">Un elemento puede desaparecer del árbol de [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] inmediatamente después de que se invoque.</span><span class="sxs-lookup"><span data-stu-id="6ec12-131">An element can disappear from the [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] tree immediately upon being invoked.</span></span> <span data-ttu-id="6ec12-132">La solicitud de información del elemento que proporciona la devolución de llamada de evento puede producir un error como resultado.</span><span class="sxs-lookup"><span data-stu-id="6ec12-132">Requesting information from the element provided by the event callback may fail as a result.</span></span> <span data-ttu-id="6ec12-133">La solución recomendada es la captura previa de la información almacenada en caché.</span><span class="sxs-lookup"><span data-stu-id="6ec12-133">Pre-fetching cached information is the recommended workaround.</span></span>

- <span data-ttu-id="6ec12-134">Los controles pueden implementar varios patrones de control.</span><span class="sxs-lookup"><span data-stu-id="6ec12-134">Controls can implement multiple control patterns.</span></span> <span data-ttu-id="6ec12-135">Por ejemplo, el control color de relleno de la barra de herramientas de Microsoft Excel implementa los <xref:System.Windows.Automation.InvokePattern> <xref:System.Windows.Automation.ExpandCollapsePattern> patrones de control y.</span><span class="sxs-lookup"><span data-stu-id="6ec12-135">For example, the Fill Color control on the Microsoft Excel toolbar implements both the <xref:System.Windows.Automation.InvokePattern> and the <xref:System.Windows.Automation.ExpandCollapsePattern> control patterns.</span></span> <span data-ttu-id="6ec12-136"><xref:System.Windows.Automation.ExpandCollapsePattern> expone el menú y el elemento <xref:System.Windows.Automation.InvokePattern> rellena la selección activa con el color elegido.</span><span class="sxs-lookup"><span data-stu-id="6ec12-136"><xref:System.Windows.Automation.ExpandCollapsePattern> exposes the menu and the <xref:System.Windows.Automation.InvokePattern> fills the active selection with the chosen color.</span></span>
